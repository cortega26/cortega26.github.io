- **`target="_blank"` without `rel=noopener`** — false positive. The one link
  the line-based scan flagged (`src/components/ContactSection.astro:169`) has
  `rel="noopener noreferrer"` on the following line. All 26 external links are
  safe.
- **Duplicated Formspree submit logic** (`public/assets/js/contact-section.js`
  vs `public/assets/js/intake-form.js`) — near-identical async-submit + button
  state, but both work and the value of consolidating is low. Not worth the
  churn.
- **Build fetches the GitHub API** (`scripts/fetch-github-stats.js`, run by
  `npm run build`) — has a fallback-on-failure and does not break the build
  offline. Resilient; no action.
- **`itemListElement`-on-`Person` as a "leave it"** — rejected in favor of
  fixing it (plan 002); it is a genuine validity defect, not cosmetic.
- **Migrating the flagship web-technical-hygiene pages onto `ServicePage` now**
  — real debt but HIGH risk / L effort, and premature: the bespoke page carries
  content the template does not model, so it needs a `ServiceContent` extension,
  not a mechanical move. Gated behind a characterization net (plan 003) instead
  of planned as a refactor now.
